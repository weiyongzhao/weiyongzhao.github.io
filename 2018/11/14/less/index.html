less的简介

less的产生

作为一门标记性语言，CSS 的语法相对简单，对使用者的要求较低，但同时也带来一些问题：CSS需要书写大量看似没有逻辑的代码，不方便维护及扩展，不利于复用，尤其对于非前端开发工程师来讲，往往会因为缺少 CSS 编写经验而很难写出组织良好且易于维护的 CSS 代码，造成这些困难的很大原因源于 CSS 是一门非程序式语言，没有变量、函数、SCOPE（作用域）等概念。LESS 为 Web 开发者带来了福音，它在 CSS 的语法基础之上，引入了变量，Mixin（混入），运算以及函数等功能，大大简化了 CSS 的编写，并且降低了 CSS 的维护成本，就像它的名称所说的那样，LESS 可以让我们用更少的代码做更多的事情。

less的安装

  1. 安装less需要下载nodejs下载地址
  2. 下载安装nodejs后使用cmd命令node -v查看版本
  3. 基于nodejs安装less：使用cmd命令npm install -g less

less的编译

​ 本质上，LESS 包含一套自定义的语法及一个解析器，用户根据这些语法定义自己的样式规则，这些规则最终会通过解析器，编译生成对应的 CSS 文件。LESS 并没有裁剪 CSS 原有的特性，更不是用来取代 CSS 的，而是在现有 CSS 语法的基础上，为 CSS 加入程序式语言的特性。

less的使用

在浏览器中对less文件进行预览

在不使用编辑器的插件的情况下在浏览器中对less文件进行预览（要刷新页面），使用less.js的过程中不能以文件形式打开，要在http://形式打开。因为less是去异步加载less源码，js解析出css代码，追加到头部的style。

  1. 需要下载less.js插件，下载地址

  2. 引入下载的less.js文件

  3. 在引入less文件的时候要进行引入类型的声明

也可以在不刷新页面的情况下对less文件进行实时的预览（热更新），使用less.js提供的watch方法，需要加入以下代码

<script>less.watch()</script>

以上使用方法解析的时间长，只适用于开发阶段的预览，上线的环境还是要使用css

mixin混入

  1. 样式混入

    在样式中混入其他的样式，在样式中调用要使用的类选择器、id选择器，这种使用方法会把公共的类或者id都编译到css选择器中，会造成代码的冗余。

  1. 函数混入

    在类选择器或者id选择器后边添加一个()后就转换成了函数，就无法将代码编译到css中了，可以在()中添加参数就可以让方法得到高度复用了。

    注意：

    1. 如果函数有参数但是没有默认值，调用的时候必须传参。
    2. 如果给了参数默认值，调动的时候传不传参数都可以了
