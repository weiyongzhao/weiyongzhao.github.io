git的基本使用介绍

git产生的背景

版本控制系统可以记录下文件的每一次修改，通过控制系统可以恢复到任意版本。有些编辑器也有版本管理的功能。

但是这些只适用于本地的版本控制，要实现多人协同开发的功能就要通过服务器来提供版本控制。

集中式版本控制系统把文件的版本都放到服务器上使所有的用户都能访问的文件的版本记录。但是缺点是版本记录太依赖服务器，服务器出现问题所有的版本都会丢失。并且如果局域网使用的话就对用户的工作场所有所限制。

分布式版本控制系统则不需要中央服务器，每个协同开发者都拥有一个完整的版本库。这样一来，任何协同开发者用的服务器发生故障，事后都可以用其他协同开发者本地仓库恢复。由于版本库在本地计算机也就不再受网络影响了。如果要将本地的修改推送给其他协同开发者还需要一台共享服务器，开发者通过这台共享服务器同步和更新数据。 分布式管理系统的代表就是Git。 git是当前最先进的代码管理工具。

git安装

window安装

Linux安装:

  • CentOS发行版：sudo yum install git
  • Ubuntu发行版：sudo apt-get install git

Mac安装

  • 打开Terminal直接输入git命令，会自动提示，按提示引导安装即可。

检测git是否安装完成可以右键看是否有Git Bash Here（命令行操作）、Git GUI Here（图形化操作），如果已经存在说明已经安装成功了。

git介绍

git工作原理

Git管理文件的3种状态：已提交、已修改和已暂存。即代表Git项目的三个工作区域的概念：Git仓库、工作目录以及暂存区域。

工作的过程：把本地的工作目录的文件提交到暂存区，此时还没有产生版本号，可以对文件进行修改，还可以进行恢复操作。确认无误后再把暂存区提交到本地仓库，此时暂存区就不存在文件了，并产生版本号。

分支

使用场景：当你正在专注开发一个功能的时，突然有一个紧急的BUG需要进行修复，我们希望保存当前任务进度再去修改BUG，等BUG修改完成后再继续我们的任务。

当我们初始化仓库的时候，Git会默认创建一个master的分支，并且有指针指到末端，指针用来表明当前处于哪个分支的哪个版本。

分支使用思路：先切换到需要修改的版本–>创建一个新的分支(此时就是需要修改的 版本)–>切换到新分支上–> 进行修改–>切换到主分支–>切换到之前进行的版本–>合并刚才进行修改所在的分支

Git常见操作

  1. 配置用户：git config --global user.name "用户名" git config --global user.email "邮箱地址"

    用于记录开发者信息，以便在版本控制记录开发者的操作行为

  2. 初始化仓库：git init

    创建一个名为.git的隐藏目录，这个目录就是存储历史版本的仓库。

  3. 查看当前目录下文件的状态：git status 如果文件名称的颜色是红色则表示文件处于已修改的状态，文件处于文件目录。如果文件名称是绿色的则表示是处于已暂存状态。如果没有任何可以提交的东西说明文件目录和暂存区没有任何可以提交的文件。

  4. 把文件从文件目录提交到暂存区：git add 文件路径 如果想把所有的文件都提交到暂存区有三种方法git add * 或者 git add .或者 git add -A

  5. 把文件从暂存区恢复到文件目录：git checkout 文件路径 如果想把所有文件都从暂存区恢复到文件目录 git checkout * 或者git checkout . 或者 git checkout -A

  6. 从暂存区提交到本地仓库：git commit -m '备注信息' 提交后会出现对应的版本号。

  7. 查看版本号：git log 或者 git reflog 会生成版本号的唯一标识，git log 只是查看当前版本之前的版本，当前版本之后的版本不会显示。git reflog 是查看所有的版本历史记录。

  8. 切换版本：git reset --hard 版本号 一般只取版本号的前边的几位就可以了。

  9. 新开分支：git branch 分支名称

  10. 查看分支：git branch 其中绿色的表示当前所处的分支。

  11. 切换分支：git checkout 分支名称

  12. 合并分支：git merge 要合并的分支

github的使用

  1. 创建仓库
  2. 把本地文件推到个人仓库：git push 仓库地址 仓库地址可以是http形式的（要进行账号、密码的验证），也可以使用SSH形式，
  3. 从个人仓库拉到本地电脑：git clone 仓库地址 （本地电脑没有仓库）、git pull 仓库地址
